01 · Overview

Two packages, one bot

ZenVPS ships as two separate downloads. The installer sets up the host — dependencies, Docker images, systemd services, and the setup wizard. The files package is the actual bot source that the installer clones or copies into place. Most people never touch the files package directly; the installer handles it automatically.

02 · Structure

Package layout

📦 ZenVPS-Installer/

  • tests/ — pytest suite (48 tests)
03 · Before you start

Prerequisites

  • A Linux host running Ubuntu 20.04 / 22.04 / 24.04 LTS or Debian 11 / 12 — the installer refuses to run on anything else (CentOS, Fedora, Alpine, Arch, etc.)
  • Root access — the installer uses sudo
  • Internet access — by default it git clones the bot source from GitHub
  • A Discord bot token from the Discord Developer Portal
  • Your Discord user ID (right-click yourself in Discord → Copy ID)
04 · Installation

One-click install

The installer is self-sufficient — run this single line as root on a fresh Ubuntu or Debian host and it takes care of everything, ending with the interactive setup wizard.

apt update -y && apt upgrade -y && apt install git -y && git clone https://github.com/Zentrix-Dev/ZenVPS-Installer.git && sudo bash ZenVPS-Installer/install.sh install

When it finishes, confirm it's alive:

sudo zenvps status

Manual install

06 · Installation

Where the bot source comes from

By default the installer clones https://github.com/ZentrixDev/ZenVPS-Files.git. Three ways to override that:

Environment variable

For non-interactive installs, point it at a fork before running:

sudo ZENVPS_REPO_URL=https://github.com/your-fork/ZenVPS-Files.git bash install.sh install

Interactive prompt

Running install.sh install in a terminal asks you to choose between git-clone and local-copy.

Local copy (offline installs)

Extract ZenVPS-Files-v1.0.0.zip next to the installer — or set ZENVPS_SRC=/path/to/python — then choose "Use a local copy" when prompted.

Later, install.sh update runs git pull on the same repo to fetch the latest version automatically.

07 · Installation

What the installer actually does

Nine steps, cleared between each for a readable flow. If any step fails, an ERR trap rolls back everything created so far, in reverse order.

  1. Pre-flight checks

    Verifies root, detects the OS, exits immediately if it isn't Ubuntu LTS or Debian stable.

  2. Install system dependencies

    apt-get install of ca-certificates, curl, wget, git, jq, sqlite3, python3, docker, and friends.

  3. Create service user + directories

    useradd --system zenvps, plus /opt/zenvps, /etc/zenvps, /var/lib/zenvps, /var/log/zenvps, /var/lib/zenvps/backups.

  4. Acquire source code

    Clones (or copies) the bot source into /opt/zenvps/.

  5. Install the Python runtime

    Creates a venv at /opt/zenvps/.venv and installs requirements.txt.

  6. Build VPS Docker images

    Builds zenvps/ubuntu:22.04 and zenvps/debian:12 from the hardened Dockerfiles.

  7. Set up configuration

    Writes /etc/zenvps/env (mode 0600, root-owned) from .env.example, then runs the setup wizard.

  8. Install systemd service

    zenvps.service, the watchdog service/timer, the anti-abuse service, logrotate config, and the CLI tool.

  9. Run health checks & start

    Verifies Docker, network, and config file — then systemctl start zenvps.

12 · Using the bot

Architecture

Slash commands flow through a permission checker into cogs, which read and write a locked JSON database and drive Docker through async subprocess calls.

Discord WebSocket │ slash commands ▼ ZenVPS Bot ├─ Permission Checker (RBAC) └─ Cogs → user_cmds · admin_cmds · remote_cmds · meta_cmds │ ├──▶ Database (zenvps.json · fcntl lock · atomic tmp+rename) │ └──▶ DockerManager (docker CLI via async subprocess) │ ▼ Docker daemon (/var/run/docker.sock) │ ▼ VPS containers (Ubuntu / Debian)

15 · Security

Security model

⛔ No --privileged

Every VPS container runs with --cap-drop=ALL plus a curated --cap-add allowlist.

⛔ No shell=True

All subprocess calls use create_subprocess_exec with argument lists, never a shell string.

⛔ No pickle

Backups are gzipped JSON, restored via json.loads() — never deserialized code.

🔑 No hardcoded credentials

Tokens and IDs are read from environment variables only.

🧹 Secret scrubbing

The logger auto-redacts Discord tokens, passwords, and SSH keys before they hit disk.

✅ Input validation

Every value crossing a trust boundary passes through security.py.

🎚️ Permission checks

Bit-flag RBAC — PermissionChecker.can(user, Permission.CREATE_VPS).

🐳 Container hardening

--memory, --cpus, --pids-limit=512, --security-opt=no-new-privileges.

⚠

If you previously used any of the original VPS-bot projects, revoke their Discord tokens immediately at the Discord Developer Portal — they were committed in plaintext in those projects. ZenVPS never writes tokens to source code; they go straight to /etc/zenvps/env (mode 0600).

16 · Security

Anti-abuse — four layers

  1. 01

    Container-level process scan

    Every 5 minutes, ps aux runs inside each VPS and is matched against a curated miner regex with word boundaries. Three strikes → suspend.

  2. 02

    cgroup CPU throttle

    Every container gets a --cpus ceiling from CPU_THROTTLE_PERCENT.

  3. 03

    Host-level anti-abuse.sh

    A separate systemd service scans host processes with pgrep, independent of any single container.

  4. 04

    Network egress filter

    Optional nftables/iptables rules blocking 50+ known mining-pool domains.

Killing a legitimate process? Edit /var/lib/zenvps/anti-abuse.sh and add it to the WHITELIST_RE regex, then restart the service — see Troubleshooting.

17 · Help

Troubleshooting

The installer couldn't clone the bot source. Possible causes:

  • The default repo URL doesn't exist yet — the ZenVPS-Files source needs to be pushed there first
  • No network access to github.com — check firewall / DNS
  • You're using a private fork — set ZENVPS_REPO_URL with embedded credentials, or configure git SSH keys

Offline fallback: extract ZenVPS-Files-v1.0.0.zip next to the installer, re-run, and choose "Use a local copy." Or set ZENVPS_SRC=/path/to/python explicitly.

The installer only supports Ubuntu 20.04 / 22.04 / 24.04 and Debian 11 / 12. Anything else exits at the pre-flight check.

The installer installs Docker via get.docker.com. If your network blocks that, install it manually first:

sudo apt-get install -y docker.io
sudo systemctl enable --now docker

Then re-run the installer.

sudo journalctl -u zenvps -n 50 --no-pager

Common causes:

  • DISCORD_TOKEN is empty → sudo zenvps wizard
  • OWNER_IDS is empty → same
  • Docker daemon not running → sudo systemctl start docker

Edit /var/lib/zenvps/anti-abuse.sh and add your process name to the WHITELIST_RE regex, then restart:

sudo systemctl restart zenvps-anti-abuse.service
18 · Help

Uninstall

sudo bash install.sh uninstall

🗑 Removed

  • /opt/zenvps/
  • All systemd units
  • /etc/logrotate.d/zenvps
  • /usr/local/bin/zenvps
  • zenvps/ubuntu:* and zenvps/debian:* images

🛡 Preserved

  • /var/lib/zenvps/ — database, keys, backups
  • /etc/zenvps/ — config
  • /var/log/zenvps/ — logs
  • The zenvps system user

Remove those manually if you want a completely clean host.
